Paso a paso

Guia de Onboarding

Todo lo que necesitas para configurar tu entorno de desarrollo en Geniova Technologies, desde los prerrequisitos hasta tu primer pnpm dev.

Antes de empezar

Prerrequisitos

Necesitas estas herramientas instaladas y configuradas en tu maquina antes de ejecutar el bootstrapper.

Nd

Node.js

>= 18

Instalar desde nodejs.org o usando nvm para gestionar multiples versiones.

pm

pnpm

npm install -g pnpm

Gestor de paquetes rapido y eficiente. Obligatorio para todos los proyectos Geniova.

gh

GitHub CLI

gh auth login

Instalar desde cli.github.com. Debe estar autenticado con acceso a la org Geniova-Technologies.

fb

Firebase CLI

npm install -g firebase-tools

Herramientas de Firebase. Autenticar con firebase login usando la cuenta Geniova.

cc

Claude Code

npm install -g @anthropic-ai/claude-code

Agente de IA para desarrollo. Requiere licencia proporcionada por Geniova Technologies.

kj

Karajan

npm install -g karajan-code

Orquestador de agentes IA. Requiere Claude (obligatorio). Codex y Gemini son opcionales (cuentas personales gratuitas).

Configuracion

Pasos de instalacion

Un unico comando inicia todo el proceso. Cada paso se ejecuta de forma secuencial y verifica el resultado antes de continuar.

1

Ejecutar el bootstrapper

El punto de entrada publico verifica que tu maquina cumple todos los prerrequisitos, valida la autenticacion con GitHub y Firebase, y confirma la pertenencia a la organizacion Geniova-Technologies.

Terminal
$ npx @geniova/developer-kit init
 
✓ Node.js v22.14.0
✓ pnpm 10.6.2
✓ gh authenticated
✓ firebase authenticated
✓ Miembro de Geniova-Technologies
2

Seleccionar proyecto

El wizard muestra los 19 proyectos disponibles organizados por categoria. Selecciona el proyecto en el que vas a trabajar.

  • Plataforma dental — 6 proyectos (Extranet V2, Intranet, Intranet-Extended, Visor-Editor, Cinema4D, Geniova Platform)
  • Herramientas internas — 10 proyectos (PlanningGame, Planning Game MCP, Portal de Incidencias, Auth&Sign, Inventario, Geniova Space, Geniova Link, Firestore-Myadmin, Web Geniova, Geniova Employee)
  • Gestion y soporte — IT, IA, Desarrollos en SAP, Marketing
Terminal
? Selecciona categoria:
> Herramientas internas
 
? Selecciona proyecto:
> Geniova Employee
3

Clonar e instalar

El wizard clona el repositorio en el directorio estandar y ejecuta la instalacion de dependencias automaticamente.

  • Clona el repo en ~/ws_geniova/{proyecto}
  • Ejecuta pnpm install para instalar todas las dependencias
  • Para proyectos con autenticacion, instala el SDK @geniova/auth desde el repositorio local
4

Configurar MCPs

Los MCPs (Model Context Protocol Servers) conectan las herramientas de IA con los servicios del proyecto. Se instalan en la configuracion de Claude Code.

Obligatorios (automaticos)
Planning Game Gestion de proyectos XP
Karajan Orquestacion de agentes y code review
Opcionales (propuestos al usuario)
Chrome DevTools Verificacion visual, screenshots, debugging
SonarQube Analisis de calidad de codigo
5

Git Hooks y Guidelines

Se ejecuta el paquete de hooks que protege el repositorio con checks automaticos e instala guidelines para todos los agentes de IA soportados.

Terminal
$ npx @geniova/git-hooks init
 
✓ Husky instalado (3 hooks)
  pre-commit, commit-msg, pre-push
✓ Guidelines generados (5 agentes)
  CLAUDE.md, .cursorrules, copilot, gemini, codex
✓ GitHub Action no-ai-refs instalada
6

Configuracion

El wizard genera el fichero .env con las variables de entorno del proyecto. Revisa los valores y completa los que aparezcan como pendientes.

  • Genera .env con variables del proyecto (API keys, Firebase config, endpoints)
  • Muestra un resumen con el estado de la configuracion y los proximos pasos recomendados
7

Verificar

Comprueba que todo funciona correctamente ejecutando el servidor de desarrollo, los tests y el linter.

Terminal
$ cd ~/ws_geniova/{proyecto}
$ pnpm dev       # Servidor de desarrollo
$ pnpm test      # Tests
$ pnpm lint      # Linting
Dia a dia

Flujo de trabajo diario

Una vez configurado el entorno, este es el flujo que seguiras para cada tarea o bug. Todas las tareas pasan por Planning Game y Pull Request.

01 Crear / asignar card
→
02 Crear rama
→
03 Implementar
→
04 Push + PR
→
05 Review
→
06 Merge
1

Crear o asignar card en Planning Game

Toda tarea debe tener una card antes de empezar a trabajar. Puede ser una task o un bug. Si no existe, crearla con titulo, descripcion estructurada y criterios de aceptacion.

2

Crear rama

Siempre trabajar en una rama dedicada. Nunca hacer push directo a main.

  • Features: feat/PRJ-TSK-XXXX-descripcion
  • Bugs: fix/PRJ-BUG-XXXX-descripcion
3

Implementar con commits pequenos

Seguir Conventional Commits (feat:, fix:, refactor:, test:, docs:, chore:). Ejecutar tests despues de cada cambio significativo. Maximo 300 lineas por PR.

4

Push y crear Pull Request hacia main

La PR debe ser atomica: un solo proposito. Incluir descripcion clara de los cambios y referencia a la card del Planning Game.

5

Review por BecarIA y companero

Karajan ejecuta el review en local y BecarIA publica el resultado como GitHub App. Un companero del equipo debe aprobar tambien antes del merge.

6

Merge tras aprobacion

Con la aprobacion de BecarIA y un companero, y el CI en verde, la PR se mergea a main. Si tiene el label automerge, se mergea automaticamente.
